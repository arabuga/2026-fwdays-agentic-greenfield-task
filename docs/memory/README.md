# Memory Bank (`docs/memory/`)

Short **working memory** for humans and agents. Canonical detail lives in
`openspec/specs/`, `docs/requirements.md`, `docs/product-brief.md`, and
`docs/qa/` as needed.

Project Factory handoff (phase, gates, slices): [`docs/current-state.md`](../current-state.md).

## Files (suggested read order)

| File | Purpose |
|------|---------|
| [`projectbrief.md`](projectbrief.md) | What the repo is for, goals, coarse layout. |
| [`productContext.md`](productContext.md) | Users, scenarios, product vocabulary. |
| [`techContext.md`](techContext.md) | Stack, versions, commands, tooling. |
| [`systemPatterns.md`](systemPatterns.md) | Architecture, behavior, constraints (operational summary). |
| [`activeContext.md`](activeContext.md) | Current focus and where to record updates. |
| [`progress.md`](progress.md) | Status, done, pending. |
| [`decisionLog.md`](decisionLog.md) | Chronological log; ADRs in `docs/adr/` when present. |

Do not duplicate long narratives here — link to specs and QA docs instead.
